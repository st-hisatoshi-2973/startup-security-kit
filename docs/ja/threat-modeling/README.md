# 脅威モデリング

このディレクトリでは、**脅威モデリング**に関するドキュメントを扱います。

脅威モデリングは、システムにおいて **何が起きる可能性があるのか** を整理し、その上でどのように守るべきかを考えるための出発点となる考え方です。

実装や具体的な設定にフォーカスしたセキュリティパターンとは異なり、**セキュリティ設計の前提となる「リスクの整理」** を目的としています。

---

## このディレクトリで扱う内容

脅威モデリングでは、次のような観点を扱います。

* どのような攻撃や不正が起きる可能性があるか
* 信頼境界がどこにあるか
* 入力・権限・データの流れがどのようになっているか
* 正常な使い方だけでなく、不正な利用がどのように成立するか

重要なのは、特定のフレームワークを使うことではなく、**設計の前にリスクを具体的に整理すること**です。

---

## バックエンドパターンとの関係

`secure-backend-patterns` では、主に

**どのように安全に設計・実装するか**

を扱っています。

例えば：

* API設計
* トークン・シークレット管理
* ログ / 監査
* 依存関係やCI/CDのセキュリティ

一方で脅威モデリングは、役割が異なります。

```
脅威モデリング → 何が起きるか
バックエンドパターン → どう守るか
```

つまり、

* 脅威モデリングは **何を守るべきか / なぜ守るのか** を明確にする
* バックエンドパターンは **どのように守るのか** を設計する

という関係になります。

---

## 使い方

典型的な流れは次の通りです。

```
1. 脅威を洗い出す（脅威モデリング）
2. 防御を設計する（バックエンドパターン）
3. 実装・検知・対応する（ログ / 監査など）
```

この流れにより、

* 想定に基づいた設計ができる
* 後付けではなく、意図的にセキュリティを組み込める

ようになります。
