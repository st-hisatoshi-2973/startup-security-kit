# AI Gateway

このディレクトリでは、**AI Gateway** に関するドキュメントを扱います。

AI Gateway は、外部のモデルAPI（LLM等）とのやり取りを制御することで、**AIの利用をどのようにシステムに組み込むかを設計するための考え方**です。

実装や設定にフォーカスしたセキュリティパターンとは異なり、**AI利用をどのように制御するかというセキュリティ設計** を扱います。

---

## このディレクトリで扱う内容

AI Gatewayでは、次のような観点を扱います。

* 外部のモデルAPIへのリクエストをどのように制御するか
* 入力と出力をどのように安全に扱うか
* 外部に送信されるデータをどのように管理するか
* AIの利用状況をどのように可視化・追跡するか

重要なのは、特定のツールを導入することではなく、

**AI利用を「把握し、制御できる状態」で設計すること**

です。

---

## 脅威モデリングとの関係

脅威モデリングは、 **何が起きるか**を整理します。

AI Gatewayは、 **どこで・どのように制御するか**を定義します。

つまり、

* 脅威モデリングでリスクを整理する
* AI Gatewayで制御ポイントを定義する

という関係になります。

---

## 設計での位置づけ

AI Gatewayは、特定の工程として使われるものではなく、

**AI利用における制御レイヤー**

として設計に組み込まれます。

そのため、

* 入力処理
* 外部通信
* 出力の利用
* ログ / 監査

といった設計の中で、制御ポイントとして機能します。
