# 開発者向けセキュリティチェックリスト

このチェックリストは、アプリケーション開発において基本的なセキュリティ対策を実装するためのガイドです。

スタートアップや小規模チームが、複雑なプロセスなしに実践的なセキュリティ対策を取り入れることを目的としています。

---

# クイックチェックリスト

## 認証 (Authentication)

* [ ] 管理者アカウントでMFA（多要素認証）が有効になっている
* [ ] 共有アカウントを使用していない
* [ ] アクセストークンの有効期限が適切に設定されている

## 認可 (Authorization)

* [ ] 役割と権限が明確に定義されている
* [ ] 重要な操作にはより強いアクセス制御が適用されている

## APIセキュリティ (API Security)

* [ ] 入力値検証が実装されている
* [ ] 重要なエンドポイントにレート制限が適用されている
* [ ] エラーメッセージに機密情報が含まれていない

## シークレット管理 (Secrets Management)

* [ ] シークレットがソースコードにハードコードされていない
* [ ] シークレットが安全な管理システムに保存されている
* [ ] シークレットへのアクセスが制限されている

## ログと監視 (Logging and Monitoring)

* [ ] セキュリティ関連イベントがログに記録されている
* [ ] 機密情報がログに出力されていない
* [ ] ログが監視されている

## 依存関係セキュリティ (Dependency Security)

* [ ] 依存関係が定期的に更新されている
* [ ] 脆弱性スキャンが実行されている
* [ ] 不要な依存関係が削除されている
* [ ] アプリケーションのSBOM（Software Bill of Materials）を生成している

# 詳細チェックリスト

## 認証 (Authentication)

### 管理者アカウントでMFAを有効にする

管理者権限を持つアカウントでは、可能な限り多要素認証（MFA）を有効にします。

### 共有アカウントを使用しない

各ユーザーは個別のアカウントを使用する必要があります。

個別アカウントにすることで、

* 操作の追跡
* アクセス権の管理

が容易になります。

### アクセストークンの有効期限を適切に設定する

可能な場合、アクセストークンは短期間で失効するように設定します。

長期間有効なトークンは、漏洩した場合の影響が大きくなります。

---

## 認可 (Authorization)

### 役割と権限を明確に定義する

アプリケーションでは、役割と権限を明確に定義します。

不要に広い権限を付与しないようにします。

### 重要な操作には強いアクセス制御を適用する

重要な操作には、より強い制御を適用します。

例：

* ユーザー権限の変更
* リソース削除
* 課金設定の変更

---

## APIセキュリティ (API Security)

### 入力値検証を実装する

外部から受け取る入力はすべて検証する必要があります。

対象：

* リクエストパラメータ
* JSONボディ
* ファイルアップロード

### レート制限を適用する

レート制限は、不正利用やブルートフォース攻撃の防止に役立ちます。

対象例：

* ログイン
* パスワードリセット
* トークン発行

### エラーメッセージに機密情報を含めない

エラー応答には次のような内部情報を含めないようにします。

* スタックトレース
* データベースクエリ
* 内部システムの情報

---

## シークレット管理 (Secrets Management)

### シークレットをハードコードしない

APIキーやデータベースパスワードなどのシークレットをソースコードに直接記述してはいけません。

### シークレット管理システムを使用する

シークレットは安全な管理システムで保存します。

例：

* クラウドのシークレットマネージャー
* 環境変数

### シークレットへのアクセスを制限する

シークレットへのアクセスは、必要なユーザーやサービスのみに限定します。

---

## ログと監視 (Logging and Monitoring)

### セキュリティイベントをログに記録する

次のようなセキュリティ関連イベントをログに記録します。

* 認証試行
* 権限変更
* 管理操作

### 機密情報をログに記録しない

ログには次のような機密情報を含めないようにします。

* パスワード
* アクセストークン
* 個人情報

### ログを監視する

ログを監視することで、不審な挙動を早期に検知できます。

---

## 依存関係セキュリティ (Dependency Security)

### 依存関係を定期的に更新する

古い依存関係には既知の脆弱性が含まれている可能性があります。

可能な場合は自動更新ツールを利用します。

### 脆弱性スキャンを実施する

依存関係の脆弱性を検出するためのツールを利用します。

例：

* Dependabot
* OSV Scanner

### 不要な依存関係を削除する

依存関係を減らすことで攻撃対象領域を小さくできます。

### SBOMを生成する

アプリケーションの依存関係を把握するため、SBOM（Software Bill of Materials）を生成します。

新しい脆弱性が公開された際に、影響を受けるコンポーネントを迅速に特定できます。

---

# 補足

このチェックリストは、開発者が実践できる基本的なセキュリティ対策に焦点を当てています。

完全なセキュリティフレームワークではなく、小規模チームがアプリケーションセキュリティを改善するための出発点として設計されています。
