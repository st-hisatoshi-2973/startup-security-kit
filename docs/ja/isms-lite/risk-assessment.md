# リスクアセスメント

## 目的

本ドキュメントは、組織の情報資産に対するセキュリティリスクを評価し、適切な対策を検討することを目的とします。

---

# リスク評価

| 資産       | 脅威      | リスク | 対策     |
| -------- | ------- | --- | ------ |
| GitHub   | アカウント侵害 | 高   | MFAの導入 |
| データベース   | データ消失   | 中   | バックアップ |
| API      | 不正利用    | 中   | レート制限  |
| 管理者アカウント | 権限悪用    | 高   | アクセス監査 |

---

# リスクレベル

| レベル | 説明       |
| --- | -------- |
| 高   | 重大な影響がある |
| 中   | 業務に影響がある |
| 低   | 影響が限定的   |

---

# リスク対応

リスクが特定された場合、以下のいずれかの対応を検討します。

* リスクの低減
* リスクの回避
* リスクの受容

---

# 継続的見直し

リスクアセスメントは以下のタイミングで見直します。

* システム変更時
* 新しいサービス導入時
* 定期レビュー
