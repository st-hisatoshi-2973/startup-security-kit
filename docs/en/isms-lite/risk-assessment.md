# Risk Assessment

## Purpose

This document evaluates security risks related to organizational information assets.

The goal is to identify potential threats and define mitigation measures.

---

# Risk Evaluation

| Asset         | Threat           | Risk   | Mitigation          |
| ------------- | ---------------- | ------ | ------------------- |
| GitHub        | Account takeover | High   | Enable MFA          |
| Database      | Data loss        | Medium | Implement backups   |
| API           | Abuse            | Medium | Apply rate limiting |
| Admin Account | Privilege misuse | High   | Access monitoring   |

---

# Risk Levels

| Level  | Description                       |
| ------ | --------------------------------- |
| High   | Severe impact on the organization |
| Medium | Moderate operational impact       |
| Low    | Limited impact                    |

---

# Risk Treatment

When risks are identified, the organization should consider the following actions:

* risk reduction
* risk avoidance
* risk acceptance

---

# Review

Risk assessments should be reviewed when:

* systems are modified
* new services are introduced
* periodic security reviews are conducted
