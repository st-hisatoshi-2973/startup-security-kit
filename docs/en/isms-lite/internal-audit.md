# Internal Audit Guide

## Purpose

This document defines the internal audit process for the ISMS Lite framework.

The goal of internal audits is to verify that security policies and controls are implemented and effective.

---

# Audit Scope

The internal audit should review the following areas:

* security policy
* asset management
* risk assessment
* access control
* incident response
* backup and recovery

---

# Audit Frequency

Internal audits should be conducted regularly.

Recommended frequency:

* once per year
* after major system changes
* after significant security incidents

---

# Audit Process

The internal audit consists of four steps.

## 1. Preparation

Define the audit scope and review relevant documents.

Examples:

* security policies
* asset register
* risk assessment

---

## 2. Review

Evaluate whether security practices follow defined policies.

Examples:

* verify MFA is enabled for critical systems
* confirm backups are performed
* check access permissions

---

## 3. Findings

Document audit findings.

Findings may include:

* policy violations
* missing controls
* improvement opportunities

Example format:

| Finding         | Risk Level | Recommendation |
| --------------- | ---------- | -------------- |
| MFA not enabled | High       | Enable MFA     |

---

## 4. Follow-up

Ensure corrective actions are implemented.

Examples:

* update security policies
* improve monitoring
* strengthen access controls

---

# Audit Independence

When possible, the auditor should be different from the person responsible for the audited area.

However, in very small teams this may not always be possible.

In such cases:

* document the audit results
* perform peer review where possible

---

# Continuous Improvement

Internal audits support continuous improvement.

Audit results should be used to:

* improve security processes
* update risk assessments
* strengthen security controls
