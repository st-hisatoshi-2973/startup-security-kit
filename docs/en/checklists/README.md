# Security Checklists

This directory contains practical security checklists for developers.

The goal of this checklist is to help development teams apply basic security practices during the development process.

Unlike security policies or frameworks, this checklist focuses on **practical actions developers can take while building software**.

---

# Available Checklists

| Checklist                       | Description                                          |
| ------------------------------- | ---------------------------------------------------- |
| developer-security-checklist.md | Basic security practices for application development |

---

# How to Use

This checklist is designed to be used during development and code review.

Examples:

* Use the checklist during pull request reviews
* Use the checklist when designing new features
* Use the checklist as part of internal development guidelines

---

# Scope

This checklist focuses on developer-level security practices such as:

* authentication
* authorization
* API security
* secrets management
* logging and Monitoring
* dependency security

This checklist translates higher-level security governance (such as **ISMS Lite**) into practical actions for developers.
